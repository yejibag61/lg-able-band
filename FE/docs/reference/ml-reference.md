# ML 참고 추출

원본 문서: `최종_LG_Able_Band_개발산출물_260609.docx`

이 문서는 FE 개발 중 ML 결과가 화면에 어떻게 보일지 이해하기 위한 참고 문서다.
FE 담당자는 `ML/` 코드를 직접 수정하지 않는다.

## ML 역할

- Python ML Server 또는 룰 엔진
- 정규화된 이벤트를 생활/위험/긴급 알림으로 분류
- 긴급도 판단
- 사용자에게 표시될 알림 유형과 위험도 결정에 관여

## ML과 관련된 기능

| FR ID | 기능 | 설명 | FE 참고 |
| --- | --- | --- | --- |
| FR-011 | 이벤트 분류 요청 | 정규화된 이벤트를 룰 엔진 또는 ML 서버를 통해 생활 알림/위험 알림/긴급 알림 등으로 분류한다. | FE는 분류 결과를 알림 유형/위험도로 표시한다. |
| FR-012 | 생활 신호 알림 생성/발송 | 생활 이벤트를 사용자 접근성 프로필에 맞춰 알림으로 생성하고 발송한다. | FE는 생활 알림 카드로 표시한다. |
| FR-013 | 위험 신호 알림 생성/발송 | 위험 이벤트를 강한 진동, 화면, 음성 등으로 사용자에게 발송한다. | FE는 위험 알림을 강하게 강조한다. |
| FR-017 | 보호자 위험 알림 발송 | 사용자 미응답 또는 위험 등급 이상 상황에서 보호자에게 알림을 발송한다. | FE는 보호자 알림 화면을 제공한다. |
| FR-021 | 생활 루틴 안내 생성 | 생활 이벤트와 사용자 상태를 기반으로 후속 행동 안내를 생성한다. | FE는 후속 행동 안내 문구를 표시한다. |

## FE가 화면에 표시할 ML 결과 후보

- 알림 유형: 생활, 위험, 긴급, 위치
- 위험도: 낮음, 보통, 높음, 긴급
- 추천 후속 행동: 예를 들어 세탁 후 건조 이동 안내
- 보호자 알림 필요 여부
- 사용자 미응답 또는 긴급도 상승 여부

## FE 구현 시 주의

- ML 결과가 아직 없으면 FE는 mock 분류 결과로 화면을 먼저 만든다.
- 위험도 색상만으로 의미를 전달하지 말고 텍스트 라벨을 함께 제공한다.
- 위험/긴급 알림은 웨어러블 화면에서도 즉시 인지되도록 짧은 문구와 큰 확인 버튼을 사용한다.
- ML 판단 오류나 분류 지연이 있을 수 있으므로 조회 실패, 분류 대기, 알 수 없음 상태를 준비한다.
