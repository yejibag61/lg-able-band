# 홈 화면 구성 계획

기준 문서:

- 화면 설계: `docs/frontend/fe-screens.md`
- 디자인 레퍼런스: `docs/frontend/fe-design-reference.md`
- 화면별 API 매핑: `docs/frontend/screen-api-map.md`
- API 명세 초안: `docs/api/api-spec-draft.md`

홈 화면은 사용자가 앱을 열었을 때 가장 먼저 보는 화면이다.
LG ThinQ의 홈처럼 카드형 대시보드 구조를 참고하되, LG Able Band에서는 안전 상태, 최근 알림, 긴급 요청을 최우선으로 배치한다.

## 홈에서 바로 답해야 하는 질문

- 지금 집 안 상태가 안전한가?
- 새로 확인해야 할 위험/생활 알림이 있는가?
- Able Band와 주요 기기가 연결되어 있는가?
- 긴급 상황에서 바로 도움을 요청할 수 있는가?
- UWB 위치 안내를 바로 시작할 수 있는가?

## 기본 화면 구성

| 순서 | 영역 | 표시 내용 | 연결 화면/API |
| --- | --- | --- | --- |
| 1 | 상단 영역 | `Able Band 홈`, 현재 공간명, 알림/메뉴 아이콘 | 알림 탭, 메뉴 탭 |
| 2 | 오늘의 상태 카드 | 안전/주의/위험 상태, 마지막 업데이트, 짧은 안내 문구 | `GET /api/app/home` |
| 3 | 긴급 도움 요청 | 큰 SOS 버튼, 요청 가능/불가 상태 | `POST /api/emergency-requests` |
| 4 | 최근 알림 | 최근 알림 3개, 유형, 위험도, 시간 | `GET /api/app/home` 또는 `GET /api/alerts?limit=5` |
| 5 | 연결 기기 요약 | 연결 기기 수, 오류/주의 기기 수, 웨어러블 상태 | `GET /api/app/home` 또는 `GET /api/devices` |
| 6 | UWB 위치 안내 | 위치 안내 가능한 기기, 탐색 시작 진입 | `GET /api/uwb/targets` |
| 7 | 보호자 연결 | 주 보호자 연결 상태, 관리 진입 | `GET /api/guardians` |

## 카드별 우선순위

### 1. 오늘의 상태 카드

홈에서 가장 먼저 보여줄 카드다.

- 정상: `오늘은 안전해요`
- 주의: `확인할 알림이 있어요`
- 위험: `위험 알림이 발생했어요`
- 긴급: `긴급 상황을 확인해주세요`

필수 표시값:

- `safetyStatus.level`
- `safetyStatus.message`
- `lastUpdatedAt`
- `unreadAlertCount`

### 2. 긴급 도움 요청

접근성 관점에서 버튼은 크고 명확해야 한다.
실수 터치를 줄이기 위해 누른 뒤 확인 상태를 한 번 보여줄 수 있다.

표시 상태:

- 요청 가능
- 보호자 없음
- 전송 중
- 전송 완료
- 전송 실패

### 3. 최근 알림

사용자가 가장 최근에 놓쳤을 수 있는 생활/위험 신호를 보여준다.

표시값:

- 알림 유형: 생활, 위험, 긴급
- 위험도: 낮음, 보통, 높음, 긴급
- 제목과 짧은 메시지
- 기기명
- 발생 시간
- 확인 여부

액션:

- 카드 선택 → 알림 상세/다시 듣기 화면
- 전체 보기 → 알림 탭

### 4. 연결 기기 요약

ThinQ의 제품 카드와 빈 상태를 참고한다.
기기가 없으면 `기기를 추가하고 Able Band와 연결해보세요` 같은 빈 상태를 보여준다.

표시값:

- 연결된 기기 수
- 주의/오류 기기 수
- 웨어러블 연결 상태
- UWB 지원 기기 수

액션:

- 카드 선택 → 기기 탭
- 기기 없음 상태 → 기기 연결 화면

### 5. UWB 위치 안내

MVP에서는 세탁기 위치 안내를 우선 예시로 둔다.

표시값:

- 위치 안내 가능 여부
- 추천 대상 기기명
- 최근 사용한 대상

액션:

- `위치 안내 시작` → UWB 가전 위치 안내 화면

## 홈 상태별 처리

| 상태 | 화면 처리 |
| --- | --- |
| 로딩 | 카드 자리에 로딩 상태 표시 |
| 정상 | 안전 상태 카드와 최근 알림 요약 표시 |
| 알림 없음 | 최근 알림 카드에 빈 상태 문구 표시 |
| 기기 없음 | 기기 추가 카드와 안내 문구 표시 |
| 보호자 없음 | 긴급 요청 카드에 보호자 연결 필요 안내 |
| API 실패 | 전체 화면을 막지 않고 실패한 카드에 재시도 버튼 표시 |
| 긴급 알림 발생 | 오늘의 상태 카드와 최근 알림을 강하게 강조 |

## Mock data 기준

홈 화면 mock data는 실제 API로 교체하기 쉽도록 다음 구조를 기준으로 둔다.

```json
{
  "user": {
    "name": "사용자",
    "homeName": "우리집"
  },
  "safetyStatus": {
    "level": "SAFE",
    "message": "오늘은 안전해요",
    "lastUpdatedAt": "2026-06-10T09:30:00+09:00"
  },
  "recentAlerts": [],
  "deviceSummary": {
    "connectedCount": 2,
    "warningCount": 0,
    "wearableConnected": true,
    "uwbTargetCount": 1
  },
  "guardianSummary": {
    "hasPrimaryGuardian": true,
    "primaryGuardianName": "보호자"
  },
  "emergency": {
    "available": true
  }
}
```

## 백엔드와 먼저 맞출 것

- `GET /api/app/home`에 홈 요약을 한 번에 담을지 결정한다.
- `safetyStatus.level` 값 목록을 확정한다.
- 긴급 요청 시 보호자 없음, 전송 실패 같은 실패 사유 코드를 정한다.
- 홈에서 UWB 대상 정보를 바로 보여줄지, 기기 탭 진입 후 조회할지 정한다.
