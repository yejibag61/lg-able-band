# LG Able Band FE 요약

원본 문서: `최종_LG_Able_Band_개발산출물_260609.docx`

이 문서는 원본 `.docx`에서 프론트엔드 개발에 필요한 내용을 추출한 요약본이다.
원본 `.docx`는 이동하거나 수정하지 않고 루트에 그대로 유지한다.

## 서비스 목적

LG Able Band는 시각·청각 장애인이 집 안의 생활 신호, 가전 상태, 위험 신호를 즉시 인지하기 어렵다는 문제를 해결하기 위한 스마트홈 연동 웨어러블 서비스다.

핵심 가치는 가전, 센서, 웨어러블, UWB를 연결하여 생활 신호와 위험 신호, 위치 정보를 사용자가 인지 가능한 방식으로 변환하는 것이다.

## MVP 방향

- 세탁 완료 알림
- 냉장고 문 열림 알림
- 공기질 상태 안내
- 가스/화재 감지 알림
- SOS 호출
- 세탁기 UWB 위치 안내
- 알림 기록 조회
- 알림 다시 듣기

## 프론트엔드 역할

- React 기반 앱/웹 화면을 제공한다.
- 사용자와 보호자의 UI를 담당한다.
- 알림 확인, 접근성 설정, 기기 연동, UWB 위치 안내 화면을 제공한다.
- 백엔드 API가 확정되기 전에는 mock data로 화면과 사용자 흐름을 먼저 구현한다.

## 앱 분리

### `FE/app`

사용자의 웹/모바일 메인 앱이다.

- 홈 화면
- 접근성 설정
- 기기 연동
- 실시간 알림
- 알림 상세/다시 듣기
- 알림 이력
- 보호자 연결/보호자 알림
- UWB 위치 안내

### `FE/wearable`

웨어러블 밴드용 표시 앱이다.

- 실시간 알림 표시
- 위험/긴급 알림 표시
- 다시 듣기
- 확인 응답
- UWB 거리/진동 안내 요약

## FE 개발 시 우선 고려사항

- 접근성: 큰 버튼, 고대비, 명확한 문구, 상태 라벨을 기본으로 한다.
- 알림 인지: 색상만으로 상태를 전달하지 않고 텍스트, 음성, 진동 상태를 함께 표현한다.
- 흐름 완성도: 사용자가 버튼을 누르고, 결과를 보고, 실패 원인을 알 수 있어야 한다.
- API 미완성 대응: mock data로 먼저 화면을 완성하고, 추후 API 계약에 맞춰 교체한다.
- 예외 상태: 로딩, 에러, 빈 데이터, 권한 없음, 외부 API 오류를 화면별로 준비한다.
