# FE 디자인 레퍼런스 정리

기준 레퍼런스: 사용자가 제공한 LG ThinQ 앱 스크린샷

이 문서는 LG ThinQ 화면을 그대로 복제하기 위한 문서가 아니라, LG Able Band의 화면 설계에 참고할 UI/UX 방향을 정리한 것이다.
API 명세와 기능 요구사항은 기존 산출물을 기준으로 유지하고, 이 문서는 화면 구성과 우선순위를 잡을 때 참고한다.

## 핵심 방향

LG Able Band의 앱은 `접근성 중심의 스마트홈 알림 대시보드`로 구성한다.

- ThinQ처럼 밝은 배경, 큰 제목, 둥근 카드, 하단 탭 구조를 참고한다.
- Able Band의 핵심 기능인 알림 인지, 긴급 요청, 보호자 연결, UWB 위치 안내를 더 앞에 둔다.
- 색상만으로 상태를 구분하지 않고 텍스트 라벨, 아이콘, 크기, 문구를 함께 사용한다.
- 사용자가 첫 화면에서 오늘의 안전 상태와 최근 위험 알림을 바로 이해할 수 있어야 한다.

## ThinQ 레퍼런스에서 가져올 것

| 참고 요소 | Able Band 적용 방식 |
| --- | --- |
| 큰 상단 제목과 집 선택 영역 | `Able Band 홈` 또는 `우리집` 같은 현재 공간 표시 |
| 하단 탭 내비게이션 | `홈`, `알림`, `기기`, `메뉴` 4개 탭으로 단순화 |
| 둥근 흰색 카드 | 상태 요약, 최근 알림, 기기 상태, UWB 안내 카드에 사용 |
| 빈 상태 화면 | 기기 없음, 보호자 없음, 알림 없음 상태를 명확하게 표시 |
| 메뉴의 그룹형 목록 | 접근성 설정, 보호자 연결, 계정/도움말을 그룹으로 배치 |
| 넓은 여백과 큰 터치 영역 | 접근성 우선 UI로 유지 |

## ThinQ에서 그대로 가져오지 않을 것

- LG ThinQ 로고, 실제 브랜드 이미지, 프로모션 배너는 그대로 사용하지 않는다.
- 제품 판매, 구독, 광고성 카드 중심의 구조는 MVP에서 제외한다.
- 작은 회색 글씨만으로 중요한 상태를 표시하지 않는다.
- 알림, 위험, 긴급 상태를 단순 아이콘 하나로만 표현하지 않는다.

## `FE/app` 기본 탭 구조

| 탭 | 역할 | 포함 화면 |
| --- | --- | --- |
| 홈 | 오늘의 상태와 핵심 액션을 빠르게 확인 | SCR-001 홈 화면 |
| 알림 | 생활/위험/긴급 알림과 이력 확인 | SCR-004 실시간 알림, SCR-005 알림 상세, SCR-008 이벤트 이력, SCR-006 보호자 위험 알림 |
| 기기 | 연동 기기와 UWB 위치 안내 | SCR-003 기기 연동, SCR-007 UWB 가전 위치 안내 |
| 메뉴 | 설정과 계정성 기능 | SCR-002 접근성 설정, 보호자 연결, 도움말 |

## 홈 화면 디자인 기준

홈은 ThinQ의 카드형 홈을 참고하되, Able Band의 안전 상태를 중심으로 재구성한다.

1. 상단: `Able Band 홈` 또는 `우리집` 제목, 알림 아이콘, 메뉴/설정 진입
2. 오늘의 상태 카드: 안전/주의/위험 상태, 마지막 업데이트, 주요 안내
3. 긴급 도움 요청: 사용자가 바로 누를 수 있는 큰 버튼
4. 최근 알림 카드: 최근 생활/위험 알림 3개 정도 표시
5. 연동 기기 카드: 연결 기기 수, 오류 기기 수, 기기 탭 이동
6. UWB 위치 안내 카드: 위치 안내 가능한 기기와 시작 버튼
7. 보호자 연결 카드: 보호자 연결 여부와 관리 진입

## 접근성 UI 기준

- 주요 버튼 터치 영역은 최소 44px 이상으로 잡는다.
- 위험/긴급 상태는 색상, 문구, 아이콘, 굵기, 상태 라벨을 함께 사용한다.
- 화면 제목과 카드 제목은 한눈에 읽히는 크기로 유지한다.
- 카드 안의 문장은 짧게 쓰고, 사용자가 해야 할 다음 행동을 분명히 적는다.
- 알림 목록은 최신순을 기본으로 하고, 위험/긴급 알림은 시각적으로 더 강하게 표시한다.
