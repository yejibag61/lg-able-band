# FE API 및 데이터 필요사항

원본 문서: `최종_LG_Able_Band_개발산출물_260609.docx`

이 문서는 FE 개발자가 백엔드/ML 담당자와 API 계약을 맞출 때 필요한 데이터 목록을 정리한다.
백엔드 구현 지시가 아니라 FE 화면 구현에 필요한 입력/출력 기준이다.

백엔드와 공유할 때는 화면별로 정리된 `docs/api/api-spec-by-screen.md`를 함께 참고한다.

## 탭 기준 데이터 묶음

LG ThinQ 레퍼런스를 반영한 `FE/app`은 하단 탭 구조를 기준으로 데이터를 묶어 사용한다.
백엔드 API는 기능별로 나뉘어 있어도, FE 구현에서는 아래 기준으로 필요한 데이터를 가져온다.

| 탭 | 필요한 데이터 묶음 |
| --- | --- |
| 로그인 | 계정 역할, access token, 사용자/보호자 프로필 요약 |
| 홈 | 사용자 요약, 오늘 안전 상태, 최근 알림, 연결 기기 요약, 긴급 요청 가능 여부, 보호자 연결 여부 |
| 알림 | 알림 목록, 알림 상세, 확인 상태, 다시 듣기 문구, 이벤트 이력 |
| 기기 | 연결 기기 목록, 기기 연결 상태, UWB 지원 여부, UWB 탐색 상태 |
| 메뉴 | 접근성 프로필, 알림 선호, 보호자 목록, 계정성 설정 |

## FE가 필요로 하는 주요 데이터

| 화면 | 필요한 데이터 | 관련 객체 |
| --- | --- | --- |
| 홈 화면 | 사용자 요약, 최근 알림, 기기 상태, 긴급 요청 가능 여부 | USER, DEVICE, ALERT, EMERGENCY_REQUEST |
| 접근성 설정 화면 | 장애 유형, 알림 선호, 보호자 연결 방식 | USER, USER_GUARDIAN |
| 기기 연동 화면 | 연동 가능 기기 목록, 연결된 기기 목록, 연결 상태 | DEVICE |
| 실시간 알림 화면 | 알림 목록, 알림 유형, 위험도, 발생 위치, 발생 시간, 확인 상태 | ALERT, DEVICE_EVENT |
| 알림 상세/다시 듣기 화면 | 상세 문구, 기기명, 발생 시간, 음성 안내 문구, 확인 상태 | ALERT, USER_FEEDBACK |
| 보호자 연결 화면 | 보호자 목록, 연락처, 주 보호자 여부, 연결 상태 | GUARDIAN, USER_GUARDIAN |
| 보호자 위험 알림 화면 | 위험 유형, 사용자 상태, 발생 시간, 확인 상태, 연락 가능 정보 | ALERT, GUARDIAN |
| 긴급 요청 | 요청 생성 결과, 전달 상태, 실패 사유 | EMERGENCY_REQUEST |
| 이벤트 이력 화면 | 기간별 이벤트 목록, 알림 이력, 상세 정보 | DEVICE_EVENT, ALERT |
| UWB 위치 안내 화면 | 대상 기기, UWB 지원 여부, 거리, 신뢰도, 탐색 상태 | DEVICE, UWB_NAVIGATION_SESSION |

## FE 기준 API 후보

실제 URL은 백엔드 담당자와 확정해야 한다.
아래 항목은 FE가 필요로 하는 기능 단위 목록이다.

| 기능 | 요청 목적 | FE에서 필요한 응답 |
| --- | --- | --- |
| 로그인/회원가입 | 사용자/보호자 인증 | 계정 역할, 사용자 또는 보호자 정보, 접근 토큰 또는 세션 상태 |
| 내 프로필 조회 | 접근성 설정 초기값 표시 | 사용자명, 장애 유형, 알림 선호 |
| 접근성 설정 저장 | 설정 변경 | 저장 성공 여부, 갱신된 프로필 |
| 홈 요약 조회 | 홈 화면 표시 | 상태 카드, 최근 알림, 기기 상태 |
| 기기 목록 조회 | 기기 연동 화면 표시 | 기기명, 유형, 연결 상태, UWB 지원 여부 |
| 기기 연결/해제 | Mock 또는 실제 연동 | 연결 결과, 실패 사유 |
| 실시간 알림 조회 | 알림 화면 표시 | 알림 목록, 유형, 위험도, 발생 시간, 위치 |
| 알림 상세 조회 | 상세 화면 표시 | 상세 문구, 기기명, 음성 안내 문구 |
| 알림 확인 처리 | 확인 완료 | 확인 상태, 응답 시간 |
| 알림 다시 듣기 | 음성 안내 재생 | 음성 안내 텍스트 또는 오디오 참조 |
| 보호자 목록 조회 | 보호자 연결 화면 표시 | 보호자명, 연락처, 주 보호자 여부 |
| 보호자 등록/수정 | 보호자 관리 | 저장 결과, 연결 상태 |
| 긴급 요청 생성 | SOS 실행 | 요청 ID, 전달 상태, 실패 사유 |
| 이벤트 이력 조회 | 이력 화면 표시 | 기간별 이벤트/알림 목록 |
| UWB 대상 조회 | 위치 안내 대상 선택 | 대상 기기 목록, 지원 여부 |
| UWB 탐색 시작 | 거리 안내 시작 | 세션 ID, 초기 거리, 상태 |
| UWB 거리 갱신 조회 | 위치 안내 화면 갱신 | 거리, 근접도, 신뢰도, 진동 패턴, 안내 문구 |
| UWB 탐색 종료 | 위치 안내 중지 | 종료 상태 |

## Mock data 작성 기준

- 실제 API 응답으로 교체하기 쉽도록 화면 안에 데이터를 직접 박지 않는다.
- 사용자, 기기, 알림, 이벤트, 보호자, UWB 세션 데이터를 분리한다.
- 알림 데이터에는 최소한 `id`, `type`, `severity`, `title`, `message`, `deviceName`, `occurredAt`, `status`를 둔다.
- UWB 데이터에는 최소한 `targetDeviceName`, `distanceM`, `confidence`, `navigationStatus`, `voiceGuide`, `vibrationPattern`을 둔다.
- 에러, 빈 데이터, 권한 없음, 미지원 상태 mock도 함께 준비한다.

## 데이터 객체 참고

| 객체 | FE에서 쓰는 의미 |
| --- | --- |
| USER | 사용자명, 접근성 유형, 알림 선호 |
| GUARDIAN | 보호자명, 연락처 |
| USER_GUARDIAN | 사용자와 보호자 연결 상태 |
| DEVICE | 기기명, 기기 유형, 연결 상태, UWB 지원 여부 |
| DEVICE_EVENT | 원천 이벤트와 발생 시간 |
| ALERT | 사용자/보호자에게 보여줄 알림 |
| ALERT_DELIVERY | 알림 전송/확인 상태 |
| EMERGENCY_REQUEST | 긴급 도움 요청 상태 |
| UWB_NAVIGATION_SESSION | 위치 안내 세션, 거리, 진행 상태 |
| USER_FEEDBACK | 알림 확인, 다시 듣기, 응답 기록 |
