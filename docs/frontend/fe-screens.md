# FE 화면 목록과 화면 설계

원본 문서: `최종_LG_Able_Band_개발산출물_260609.docx`

이 문서는 FE 개발 시 화면 단위 작업을 바로 나눌 수 있도록 메뉴구조도와 화면설계서를 Markdown으로 정리한 것이다.

## 디자인/탭 기준

LG ThinQ 레퍼런스를 참고해 `FE/app`은 하단 탭 기반 구조로 정리한다.
다만 ThinQ의 제품 관리 중심 구조를 그대로 따르지 않고, Able Band의 핵심 가치인 알림 인지, 긴급 요청, 접근성 설정, UWB 위치 안내를 우선한다.

상세 디자인 기준은 `docs/frontend/fe-design-reference.md`, 홈 화면 구성은 `docs/frontend/home-screen-plan.md`를 따른다.

## 화면 설계

| 화면 ID | 화면명 | 목적 | 진입 경로 | 표시 정보 | 주요 액션 | 예외 메시지 | 관련 FR |
| --- | --- | --- | --- | --- | --- | --- | --- |
| SCR-000 | 로그인/역할 선택 화면 | 사용자/보호자 역할별 로그인 | 앱 최초 진입 | 역할 선택, 이메일, 비밀번호 | 로그인, 역할별 화면 이동 | 계정 없음, 비밀번호 오류, 권한 없음 | UC-001 |

| SCR-003 | 기기 연동 화면 | 가전, 센서, 웨어러블 연결 | 기기 탭 | 연동 기기 목록, 연결 상태 | 기기 연결, 해제 | 외부 API 오류, 권한 없음 | FR-006, FR-007 |
| SCR-004 | 실시간 알림 화면 | 생활/위험/긴급 알림 확인 | 알림 탭 | 알림 카드, 위험도, 발생 위치 | 확인, 다시 듣기, 상세 보기 | 알림 없음, 권한 없음 | FR-012, FR-013, FR-014 |

| SCR-007 | UWB 가전 위치 안내 화면 | 대상 가전까지 거리와 도착 상태 안내 | 기기 탭 > 가전 위치 안내 | 대상 가전, 거리, 진동 상태, 음성 안내 문구 | 탐색 시작/중지, 다시 안내 | UWB 미지원, 신호 실패 | FR-023~FR-028 |
| SCR-008 | 이벤트 이력 화면 | 알림/이벤트 이력 조회 | 알림 탭 > 이력 | 기간 필터, 이벤트 목록, 상세 | 상세 보기, 다시 듣기 | 데이터 없음, 조회 실패 | FR-015, FR-020 |

## 홈 화면 구성 메모

홈 화면은 ThinQ의 카드형 대시보드를 참고하되, Able Band의 핵심 액션을 먼저 둔다.

1. 상단 제목과 현재 공간명
2. 오늘의 안전 상태 카드
3. 긴급 도움 요청 버튼
4. 최근 알림 카드
5. 연결 기기 요약 카드
6. UWB 위치 안내 카드
7. 보호자 연결 상태 카드

세부 구성은 `docs/frontend/home-screen-plan.md`를 따른다.

## `FE/app` 화면 작업 순서 추천

1. 로그인/역할 선택 화면
2. 공통 레이아웃과 하단 탭 구조
3. 홈 화면
4. 실시간 알림 화면
5. 알림 상세/다시 듣기 화면
6. 이벤트 이력 화면
7. 접근성 설정 화면
8. 기기 연동 화면
9. 보호자 연결/보호자 위험 알림 화면
10. UWB 위치 안내 화면

## `FE/wearable` 화면 작업 순서 추천

1. 기본 웨어러블 프레임
2. 실시간 알림 표시
3. 위험/긴급 알림 표시
4. 확인 응답
5. 다시 듣기
6. UWB 거리/진동 안내 요약
